- Updated AppHost to orchestrate all services with dedicated ports:

  - auth-api (port 5001)
  - users-api (port 5005)

  - orders-api (port 5003)
  - payments-api (port 5004)

### Key learnings:
- Each microservice should only use its own DbContext, not reference other services' databases
- The main API (gateway) should proxy requests to microservices via HTTP, not direct database access
- MediatR handlers need to be registered from the correct assembly
- EF Core migrations need to handle existing tables gracefully using PostgreSQL DO blocks
- HttpClient URL should be configurable via configuration instead of hardcoded
