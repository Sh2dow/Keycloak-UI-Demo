 ### Areas for Improvement (Per Refactor Plan)

 #### 1. Domain Models - Current: POCOs → Target: Rich domain models

 Current State:

```csharp
   public abstract class Order
   {
       public Guid Id { get; set; } = Guid.NewGuid();
       public Guid UserId { get; set; }
```

   **AppV2 Pattern to Mirror:**
   - Domain validation encapsulated in entities
   - Private setters with public getters
   - Domain events for side effects
   - Business logic within entities

   #### 2. **Validation Strategy** - Current: Command-level only → Target: Domain-level + Command-level
   **Current State:**
   - Validation only in `CreateOrderCommandValidator`
   - No domain model validation

   **AppV2 Pattern:**
   - Static factory methods with validation
   - Return `Result<T>` from factory methods
   - Domain rules enforced at entity level

   #### 3. **Result Pattern** - Current: Basic → Target: Enhanced with Problem Details
   **Current State:**
   ```csharp

   public sealed class Result<T> { ... }
   ```

   **AppV2 Pattern:**
   - `IntegrationResult` for cross-service errors
   - `ValidationResult` for validation-specific errors
   - Exception types (`ValidationException`, `IntegrationException`)

   #### 4. **Configuration Security** - Current: appsettings.json secrets → Target: Environment variables + Secret Manager
   **Current State:**
   - Connection strings with passwords in config
   - No visible user-secrets setup

   #### 5. **Testing** - Current: Limited visibility → Target: Comprehensive unit/integration tests
   **Missing:**
   - Unit tests for domain models
   - Integration tests for handlers
   - Database test fixtures

   ## Specific Recommendations

   ### Priority 1: Domain Model Enhancement
   
   ```csharp
   public abstract class Order
   {
       private string _status = "Pending";
       private readonly List<OrderEvent> _events = [];

       public string Status => _status;
       public IReadOnlyList<OrderEvent> Events => _events;

       protected Order() { }

       // Add domain validation factory
       public static Result<Order> Create(
           Guid userId,
           decimal totalAmount,
           string orderType,
           string? downloadUrl = null,
           string? shippingAddress = null)
       {
           var errors = new List<ResultError>();

           if (string.IsNullOrWhiteSpace(order Type))
               errors.Add(new("validation", "OrderType is required", nameof(orderType)));

           if (totalAmount <= 0)
               errors.Add(new("validation", "TotalAmount must be positive", nameof(totalAmount)));

           if (orderType == "digital" && string.IsNullOrWhiteSpace(downlo adUrl))
               errors.Add(new("validation", "DownloadUrl required for digital orders", nameof(downloadUrl)));

           if (orderType == "physical" && string.IsNullOrWhiteSpace(shippi ngAddress))
               errors.Add(new("validation", "ShippingAddress required for physical orders", nameof(shippingAddress)));

           if (errors.Any())
               return Result.Failure<Order>(errors);

           var order = new Order {
               UserId = userId,
               TotalAmount = totalAmount,
               Status = "Pending"
           };

           return Result.Success(order);
       }

       public Result UpdateStatus(string newStatus, string reason)
       {
           var currentStatus = Status;

           // Domain rules for valid transitions
           if (newStatus == "Completed" && currentStatus != "Paid")
               return Result.Failure(new ResultError(
                   "domain",
                   $"Cannot transition to Completed from {currentStatus}",
                   nameof(newStatus)));

           // Raise domain event
           _events.Add(new OrderStatusChanged(Id, newStatus, reason));
           _status = newStatus;

           return Result.Success();
       }
   }
   ```

   ### Priority 2: Domain Services
   Extract business logic from handlers:
   ```csharp
   public interface IOrderCalculator
   {
       decimal CalculateTotal(IEnumerable<Order Item> items);
       decimal ApplyDiscount(decimal total, Discount discount);
   }

   public class OrderCalculator : IOrderCalculator
   {
       public decimal CalculateTotal(IEnumerable<Order Item> items)
       {
           return items.Sum(item => item.Quantity * item.Price);
       }

       public decimal ApplyDiscount(decimal total, Discount discount)
       {
           return discount.Type switch
           {
               DiscountType.Percentage => total * (1 - discount.Value),
               DiscountType.Fixed => total - discount.Value,
               _ => total
           };
       }
   }
   ```

   ### Priority 3: Configuration Security

   ```json

     "ConnectionStrings": {
       "Default": "Host=localhost;Port=5432;Database=keycloak_demo;Username=keycloak"
   ```

   ```bash
   dotnet user-secrets init
   dotnet user-secrets set "ConnectionStrings:Default:Passw ord" "your_password"
   ```
